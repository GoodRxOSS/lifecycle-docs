After Lifecycle deploys your first Environment, confirm that its Services are
ready before you start testing. You can inspect the Environment in the
Lifecycle UI and use the comments on your pull request to change or monitor it.

## Confirm the Environment is ready

1. Follow the **View details** link in the Lifecycle comment, or open
   **Environments** in Lifecycle and select your Environment.
2. Confirm that the Environment status is **Deployed**.
3. Select each Service you plan to test. A successfully deployed Service shows
   **Ready** and the message **Kubernetes pods are ready**.
4. Select **Visit URL** to open a public Service.

If a Service shows **Build failed**, **Deploy failed**, or **Error**, open its
**Console** instead. See [Troubleshoot a failed
deployment](/docs/troubleshooting/deploy-issues) for the recovery workflow.

## Test the example application

Open the deployed `frontend` Service from **Visit URL** or from the pull request
comment, then:

1. Add and complete a few tasks to confirm the frontend can update backend data.
2. Open the `variables` page to inspect the non-secret configuration exposed by
   the example application.
3. Return to Lifecycle when you need to inspect another Service or its logs.

## Mission Control Comment

The **Lifecycle PR comment** in your pull request serves as mission control for
your Environment.

<Image
  src="/getting-started/pr-comment.png"

  width={2106}

/>

### What You Can Do in the PR Comment

- **Service checkboxes:** Select or deselect optional Services.
- **Redeploy Environment:** Queue another deployment after you correct a
  transient problem.
- **Deployment links:** Open deployed Services and, when configured, related
  dashboards.

## Status Comment

When you add the `lifecycle-status-comments!` label to the pull request,
Lifecycle adds a **status comment**.

Lifecycle updates this comment as the Environment progresses and includes the
available deployment links.

<Image
  src="/getting-started/status-comment.png"

  width={1902}

Notice the following while the environment is being built:

- The comment updates as the Environment changes state.
- Each Service reports its current status.
- Build-log links appear when the deployment backend provides them.
