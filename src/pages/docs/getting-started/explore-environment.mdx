## Quick Reference

| Component           | Purpose                            | Access                                 |
| ------------------- | ---------------------------------- | -------------------------------------- |
| **Mission Control** | Manage services, trigger redeploys | PR comment (pinned)                    |
| **Status Comment**  | Real-time build/deploy progress    | Add `lifecycle-status-comments!` label |
| **Service URLs**    | Access deployed services           | Links in PR comments                   |
| **Logs**            | Debug build/runtime issues         | Status comment or dashboard            |

## Test Your Application

1. Click the `frontend` link in the PR comment
2. Add/complete tasks to verify backend connectivity
3. Visit the `variables` page to inspect container environment
4. Confirm everything works as expected

## Deploy Status Reference

<Mermaid
  chart={`
flowchart LR
    pending --> queued --> cloning --> building --> built --> deploying --> ready
    cloning --> error
    building --> build_failed
    deploying --> deploy_failed
`}
/>

| Status          | Meaning                     |
| --------------- | --------------------------- |
| `pending`       | Waiting to be processed     |
| `queued`        | In deployment queue         |
| `cloning`       | Fetching source code        |
| `building`      | Building Docker image       |
| `built`         | Image built successfully    |
| `deploying`     | Rolling out to Kubernetes   |
| `ready`         | Healthy and serving traffic |
| `waiting`       | Waiting for dependencies    |
| `build_failed`  | Docker build error          |
| `deploy_failed` | Kubernetes deployment error |
| `error`         | Unexpected failure          |
| `torn_down`     | Service removed             |

## Viewing Logs

Access logs from the status comment or Lifecycle dashboard.

| Log Type           | Shows                                                       |
| ------------------ | ----------------------------------------------------------- |
| **Build logs**     | Dockerfile execution, dependency installation, build errors |
| **Container logs** | Application startup, runtime errors, request handling       |

## Dashboard Integration

When configured, your environment includes links to:

- **Datadog Logs** - Centralized logging
- **Fastly Dashboard** - CDN config (if enabled)
- **Custom Dashboards** - Organization-specific telemetry

## Troubleshooting

| Problem                     | Solution                                       |
| --------------------------- | ---------------------------------------------- |
| Service stuck in `building` | Check build logs for errors                    |
| `build_failed` status       | Review Dockerfile and dependencies             |
| `deploy_failed` status      | Check resource limits and health checks        |
| Service not appearing       | Verify checkbox is selected in Mission Control |
| Stale environment           | Check the Redeploy checkbox                    |
